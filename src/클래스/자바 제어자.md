
# 접근 지정자
  ## 멤버 및 생성자의 접근 지정자
    멤버 및 생성자에는 public, protected, default(또는 package), private이라는 4가지 종류의 접근 지정자 사용 가능하다.
    
    - public : 동일 패키지의 모든 클래스 + 다른 패키지의 모든 클래스에서 사용 가능
    - protectd : 동일 패키지의 모든 클래스 + 다른 패키지의 자식 클래스에서 사용 가능
    - default : 동일 패키지의 모든 클래스에서 사용 가능
    - private : 동일 클래스에서 사용 가능

  ## 클래스의 접근 지정자
    클래스에서는 public, default 접근 지정자만 사용할 수 있다.

  ## 클래스 접근 지정자와 생성자 접근 지정자의 연관성
    클래스에 생성자가 없을 때 컴파일러가 기본 생성자를 자동으로 생성하는데 이때 자동으로 추가되는 생성자의 접근 지정자는
    클래스의 접근 지정자에 따라 결정된다. 


# static 제어자

  static은 클래스 멤버(필드, 메서드, 이너 클래스)에 사용하는 제어자. 
  객체 안에 있을 때 사용할 수 있는 상태가 되는 멤버를 '인스턴스 멤버'라고 하고 쉽게 말해 멤버 앞에 static이 없다.
  반면 앞에 static이 붙어 있는 멤버를 '정적 멤버'라고 한다.
  
 ## 인스턴스 필드와 정적 필드

  인스턴스 필드와 정적 필드는 사용하는 방법이 조금씩 차이가 나는데 그 이유는 각 필드의 저장 위치 때문이다.
  인스턴스 필드를 사용하기 위해서는 객체를 생성해야만 사용 가능하고 객체 내부에 생성되므로 저장 공간이 힙 메모리에 위치
  하므로 반드시 저장 공간에 값을 읽거나 쓰기 위해서는 참조 변수명을 사용해야 한다.

  정적 필드는 클래스 내부에 공간을 지니고 있기 때문에 객체 생성 없이 바로 사용할 수 있는 것이다.


     class A {
        int m = 3;  // 인스턴스 필드 
        static int n = 5; // 정적 필드
    }

    1. 인스턴스 필드의 활용 방법 : 객체를 생성한 후에 사용 가능
    A a = new A(); -> sout(a.m);

    2. static 필드의 활용 방법 1 - 객체를 생성하지 않고 바로 사용
    sout(A.n);

    3. static  필드의 활용 방법 2 - 객체를 생성한 후 사용 (권장 X why? 정적 필드라는 걸 확실하게 나타내기 위해서) 
    A a = new A(); -> sout(a.n);

  반드시 기억해야 하는 정적 필드의 특징은 '객체 간 공유 변수의 성질'이 있다는 것이다.
 
  ## 인스턴스 메서드와 정적 메서드
    인스턴스 필드, 정적 필드와 사용법 같다

  ## 정적 메서드 안에서 사용할 수 있는 필드와 메서드

    정적 메서드 내에서는 정적 필드 또는 정적 메서드만 사용할 수 있다.
    정적 메서드는 객체 생성 없이 실행될 수있어야 하는데 인스턴스 멤버는 반드시 객체를 생성해야 사용할 수 있다.
    객체 생성 이전에 실행하려면 내부에는 객체 생성 이전에 사용할 수 있는 요소들만 구성되어 있어야 하므로
    이것이 바로 정적 메서드 내부에 정적 멤버만 올 수 있는 이유다.

  ## 정적 초기화 블록

  정적 필드는 객체 생성 이전에도 사용할 수 있어야 하므로 생성자에서는 정적 필드를 초기화할 수 없다는 말이다.
  따라서 정적 필드를 초기화하기 위한 문법을 별도로 제공하는데 이것이 정적 초기화 블록이다.

    static { 
        // 클래스가 메모리에 로딩될 때 실행되는 내용
    }

  
  ## static main() 메서드
  
  프로그램을 시작하면 가장 먼저 실행되는 main() 메서드는 왜 정적 메서드로 구성됐을까?

    자바 가상 머신이 main()메서드를 실행시켜 주기 떄문이다. main() 메서드가 정적 메서드이므로 자바 가상 머신은
    '실행 클래스명.main()'을 호출하는 것만으로도 이 메서드를 실행하는 것이다. main() 메서드가 인스턴스 메서드라면
    아무리 자바 가상 머신이라도 객체의 참조 변수를 이용해 main() 메서드를 호출해야만 했을 것이다.
    따라서 프로그램을 실행할 때 가장 먼저 실행되는 메서드인 main() 메서드를 정적 메서드로 구성한 것이다.
