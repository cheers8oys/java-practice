
# 클래스와 객체의 개념

  ## 클래스의 개념 알아보기
    객체는 사용할 수 있는 실체를 의미하며, 클래스는 객체를 만들기 위한 설계도와 같다.

    클래스를 왜 사용할까?
     프로그램 문법 요소의 발전 과정 (변수 -> 배열 -> 구조체 -> 클래스)
     다양한 자료형의 데이터를 하나로 묶어 관리할 수잇는 구조체는 말 그대로 데이터만 묶어 놓은 것이기 때문에 데이터를 
     관리하는 다양한 기능까지 함께 관리하는 문법 요소다.

  ## 절차지향과 객체지향 이해하기
    절차지향형 프로그램 : 말 그대로 이야기를 순서대로 써 나가듯이 프로그램을 구성하는 방식
    객체지향형 프로그램 : 프로그램에서 사용되는 객체를 생성하고 객체에 포함된 데이터와 기능을 상호 호출함으로 프로그램을 구성
 
  ## 자바에서 제공하는 객체지향 문법 요소
    '클래스'와 '인터페이스'가 있고 클래스는 다시 '일반 클래스'와 '추상 클래스'로 나뉜다.

  ## 클래스 구조 살펴보기
    클래스 외부 구성 : 패키지, 임포트, 외부 클래스
    클래스 내부 구성 : 필드, 메서드, 생성자, 이너 클래스

  ## 클래스와 객체 구분하기
    클래스를 붕어빵 기계에 비유한다면 객체는 붕어빵 기계로 찍어 낸 붕어빵에 비유할 수 있다.
    클래스의 생성자로 객체를 만드는 과정을 '인스턴스화'라고 하고 인스턴스화로 만들어진 객체를 '인스턴스'라고 한다.
    객체 속에는 클래스의 내부 구성 요소 중 생성자를 제외한 나머지 요소가 포하모대 있는데 이를 '인스턴스 멤버'라고 한다.
    **클래스는 바로 사용할 수 없고 반드시 객체를 생성해 객체 안에 있는 필드, 메서드 및 이너 클래스를 사용해야 한다.**
